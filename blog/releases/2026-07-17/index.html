earendil-works/pi v0.80.9 - v0.80.10 gpt-oss-120b

新内容

  • 新增 Kimi Coding 思考兼容性，模型可自适应思考并支持空签名块，K3 暴露最大支持级别
  • 新增 Kimi K3 支持，可在内置提供商中使用
  • 新增 Kimi 延迟工具加载，实现基于扩展的工具激活
  • 为 Kimi Coding、Moonshot AI、Moonshot AI China、OpenRouter…
  • 登录流程改为预填设备授权链接 “Sign in with SuperGrok or X Premium”，并将默认 xAI 模型改为 Grok 4.5

Breaking Changes

  • 移除内置 xAI 模型目录中的 Grok 3、Grok 3 Fast、Grok 4.20 以及 Grok Code Fast 1，需迁移至其他模型

问题修复

  • 修复 Kimi Coding 请求使用 Anthropic 自适应思考且不受 token 预算限制，启用空思考签名。
  • 修复 Moonshot AI 与 Moonshot AI China 的定价元数据。
  • 修复 K3 思考级别元数据仅显示支持的最大级别。
  • 恢复因 0.80.9 移除的 xAI 模型目录生成。
  • 修复 Vercel AI Gateway 与 OpenRouter 模型的 Kimi K3 输出限制
  • 修复在首次助手响应前克隆或分叉会话时未保存的提示错误

rorkai/App-Store-Connect-CLI 3.0.0 gpt-oss-120b

新内容

  • 定义标准的 view 与 edit 命令
  • 仅构建选中的命令子树以提升构建速度
  • 延迟在首次查找时注册 output-renderer
  • 当环境凭证完整时直接跳过钥匙串查找
  • 将默认上传块并发提升至 4 并优化传输性能
  • 新增 pricing 功能：创建应用可用性

问题修复

  • 修复 pricing、telemetry、subscriptions 等模块的分类错误（缺失可用性、元数据验证、状态不匹配等）
  • 修复 CLI 未知标志诊断、接受分析标志别名、帮助文本错误并稳定提示测试
  • 修复 auth 中无效私钥、无效凭证的错误分类为使用或认证错误
  • 添加分页循环保护并在写入失败时清理临时密钥
  • 保留 Web HTTP 状态退出码并在错误重试及认证失败时显示状态码
  • 修正订阅可用性修复逻辑
  • 改进发布流程：强制校验安装校验和并压缩发布二进制

nexu-io/open-design open-design-v0.15.1 gpt-oss-120b

新内容

  • OpenDesign Agent 升级，图像细节保留更完整，长会话保持连贯并可恢复
  • 多模态模型读取更细致图像，截图分析与视觉批评更可靠
  • 对话内提问保持，支持可视化选择与多选答案汇总
  • 首页明确协作模型，提升设计与代理的交互体验
  • 新增 AtomCode 代理运行时、SiliconFlow Global BYOK 预设及并行网页搜索
  • 多轮对话复用已知信息，意图信号更精准，减少不必要的提示重发
  • 默认关闭 “从聊天中学习”，记忆更干净且可自行控制
  • 静默更新提示更清晰，升级流程更易理解
  • HTML 预览更可靠，自动打开、编辑历史保留、预览刷新完整源码
  • 导出 PPT/PDF 包含全部幻灯片，编辑 PPTX 保持 CJK 字体，缩略图不再消失

问题修复

  • 修复代理中断导致任务结束并实现恢复，AMR 重试临时关闭
  • 修复 BYOK 代理错误处理，权限提示不再阻塞，提供者未找到响应干净结束
  • 修复模型名称过长显示、Codex 图像前缀接受、od 媒体生成接受提示文件等集成问题
  • 修复启动 / 更新过程中的不可靠行为，数据库二进制与运行时对齐，更新清理等待
  • 修复并发导入重复导致的服务器错误，改为去重处理并安全阻止不安全网络目标
  • 修复若干界面小回归，包括主页层、浏览器指引、表单文本、画廊默认打开所有内容类型
