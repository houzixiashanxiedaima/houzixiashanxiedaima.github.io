MHSanaei/3x-ui v3.3.1 GPT-OSS 120B

新内容

  • 实时通过 Xray gRPC API 应用 inbound/outbound/routing 配置，
  • 原生地理数据自动更新，使用 Xray‑core 内置机制
  • 使用 Xray online‑stats API 实现免解析在线统计和 IP 限额
  • 更智能的多节点同步，推送全局客户端使用量并本地强制
  • 基于 outbound 的出口桥接，面板流量通过负载均衡器转发

问题修复

  • 将 log.access 与 log.error 限制在面板日志目录，修复任意文件写入漏洞

Breaking Changes

  • 面板旧的代理 URL 被 outbound 出口桥接取代，需检查并更新配置
  • 地理数据自动更新后不再使用自定义 geo 管理器，若有手动路径需迁移

tldraw/tldraw v5.1.1 GPT-OSS 120B

问题修复

  • 修复在触摸屏进入笔模式时忘记先前触摸的问题，取消旧触摸并从笔首次触摸处开始绘制。

steipete/CodexBar v0.34.0 GPT-OSS 120B

新内容

  • 新增可选导入 GitHub 计费预算窗口并在卡片及菜单栏图标中展示预算指标
  • 新增韩语本地化支持及语言选择器
  • 新增德语作为可选应用语言
  • 新增土耳其语作为可选应用语言
  • 新增 Devin 每日和每周配额追踪，支持已登录 Chrome 会话或手动 Bearer token

问题修复

  • 修复 CLI 在决定其他提供者是否需要 macOS Web 支持时，正确限定 Ollama API
  • 修复 Provider switcher 在等宽分段会被截断时，保持本地化标签可见
  • 修复 OpenAI API 使用分页导致的组织使用量统计不足问题
  • 修复菜单栏手动刷新时提供即时反馈并保持几何位置稳定
  • 修复 Amp 在使用访问令牌时恢复使用数据获取，并保留浏览器 Cookie 解析作为回退

jxxghp/MoviePilot v2.13.8 GPT-OSS 120B

新内容

  • 更新 allowed-tools 并在 SKILL.md 中完善插件创建说明
  • 新增插件文档，帮助创建和管理 MoviePilot 插件
  • 添加后端健康检查，监控未受管的 MoviePilot 进程
  • 新增下载器不完整后缀切换功能
  • 新增 doctor 诊断自救功能

问题修复

  • 将插件备份日志级别改为 debug
  • 将插件恢复日志级别改为 debug 并添加警告过滤测试
  • 修复 TorrentLeech 最新种子列表从第 21 页开始请求的问题
  • 实现工具执行超时处理并改进阻塞调用管理
  • 改进缓存锁机制，优化文件和 redis 后端的键处理

fishjar/kiss-translator v2.0.24 GPT-OSS 120B

新内容

  • 新增 AI 词典功能，可自动检测语言并进行语音播报
  • 新增提示词管理功能，便于自定义和组织提示词
  • 新增同步加密密码支持，提升多端数据同步的安全性
  • 优化 YouTube 字幕处理，支持翻译失败字幕块即时修复并导出原始字幕为 JSON
  • 使用 Trusted Types 保护 HTML 解析，界面隐藏 API 密钥提升安全性

问题修复

  • 修复 Via 浏览器中流式 UTF-8 解码回退问题
  • 修复 GitHub Gist 同步重复问题并优化同步流程
  • 补充缺失的 connect 域名，完善桥接逻辑和请求回退处理

jdx/mise v2026.6.5 - v2026.6.6 GPT-OSS 120B

新内容

  • 新增 mise bootstrap 命令，实现端到端机器配置，可选 —yes、—dry-run、
  • 支持声明式 dotfiles，提供 add、apply、edit、status，支持 symlink
  • 新增 bootstrap.macos.defaults 用于声明式管理 macOS 用户默认设置，可
  • 新增 brew-cask 包管理器，直接从 cask API 安装应用捆绑，无需本地 Homebre
  • 在缺少 bottle 时实现原生 Homebrew 源码构建，自动 provision Ruby 并
  • 关闭多个信任绕过向量，提升安全性
  • 将 aqua 的 GitHub 证明验证与注册表要求保持一致

问题修复

  • 修复 CLI 在参数含无效 UTF-8 时不再崩溃，错误以 U+FFFD 报告
  • 改进 GitHub 403 错误信息，现显示响应体、速率限制头和身份验证状态
  • 将 Brew shim (shim.rb) 包含在发布的 crate 中，解决 cargo publ
  • 把 github、gitlab、forgejo 的 credential_command 设为全局仅
  • 修复 GHSA-436v-8fw5-4mj8 漏洞，忽略本地配置中的 ci、paranoid、tru
  • 在无本地 mise 配置的仓库中，加载默认任务目录前需先获得信任，防止 mise-tasks/ 目录
  • 在 aqua 中转发 github_artifact_attestations.predicate_
  • 统一 aqua 变量选项的规范化，重复变量将报冲突错误

Breaking Changes

  • credential_command 等仅限全局使用，项目本地配置将被剥离
  • trusted_config_paths、paranoid 等在本地配置中被忽略，可能导致行为变化
