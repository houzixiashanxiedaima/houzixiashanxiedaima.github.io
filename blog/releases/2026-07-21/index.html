NousResearch/hermes-agent v2026.7.20 gpt-oss-120b

新内容

  • 首次响应时间大幅提升，冷启动延迟从约 4.3 秒降至约 0.9 秒，约 80% 减少，适用于所有平台。
  • 桌面应用性能提升，Markdown 拆分器 CPU 使用降低 14 倍，增量块词法解析并虚拟化审阅面板。
  • TUI 增量渲染流式 Markdown，提升大段对话的流畅度。
  • 新增 Fireworks AI、DeepInfra、Upstage Solar 等模型提供商，支持费用估算和缓存选择器。
  • 引入可插拔 SecretSource 接口，支持 Bitwarden 与 1Password 等密钥管理。
  • 新增 CLI 命令 /model —once，实现一次性模型覆盖并自动恢复。
  • 会话导出支持 Markdown、QMD、HTML、prompt-only、trace 等格式，并可上传至 HuggingFace。
  • 网关功能增强：持久交付账本、基于配置文件的路由以及多路复用配置。
  • 消息平台新增内联选择器（Telegram、Discord、Matrix）及 WhatsApp 原生投票、位置、丰富元数据。
  • 桌面应用采用布局树模型的可贡献 shell，插件化 i18n 本地化支持。
  • 新增能力页面，将技能、工具、MCP 集中展示并提供响应式导航。
  • Web Dashboard 增加内存提供者切换、会话安全导入、WhatsApp 配对及 Telegram Bot 设置等功能。
  • CLI 与 TUI 新增 /subscription 与 /topup 计费命令。
  • 工具系统与 MCP 更新：新增 Blender 工具、改进 mcp__server__tool 命名、OAuth 回调端口竞争解决。
  • 统一的 reasoning_effort 与 MoA 配置，支持 per-model、per-task、per-slot 细粒度控制。

安全修复

  • Vertex 凭证 / 项目 / 区域解析通过配置文件密钥范围硬化，子进程环境变量被剥离。
  • 六项 P1 安全硬化 PR 包括浏览器防护、媒体锚定、.env 锁定、委托 ACP 传输等。
  • SecretSource 接口实现安全密钥管理，支持 Bitwarden 与 1Password。

问题修复

  • 若干回滚与未发布功能已移除，确保系统稳定性。

  • 优化决策流程：明确 Strategist（准备）与 Executor（实现）的分工，提升原生对象（图表 / 表格）的转换逻辑
  • 改进规划产物生成：恢复全文档编写模式，一次性生成完整的 design_spec.md 和 spec_lock.md
  • 增强图表检索：引入语义回退机制（semantic_fallback），在词法匹配失效时提供完整注册表审查
  • 支持自定义创意方向：spec_lock.md 现在接受可选的目录引用列表以实现自定义视觉风格
  • 优化主题研究：改为由结果关键事实缺失驱动的补缺模式，并延迟图像获取至最终确认后
  • 更新文档：重构路线图，并在 README 中推荐 Kimi K3 为默认模型，Claude 为备选
  • 优化确认机制：支持通道中立的决策确认（兼容 Confirm UI 和对话界面）
  • 增强裁剪策略：在 §VIII 中新增明确的裁剪策略列（no-crop 或 adaptive）

问题修复

  • 修复 v4 版本架构清理导致的提示词过度压缩问题，恢复 AI 的设计权限与范围
  • 修复 project_manager.py 验证逻辑，使其仅针对模式验证规划产物而非重新打开 UI
  • 修复孤立锁（Orphan locks）的重建机制，并对齐恢复指南

jdx/mise v2026.7.11 gemma-4-31b

新内容

  • 配置结构化工具定义现在统一支持 version, path, prefix 和 ref 选择器
  • 新增 mise bootstrap repos update 和 exec 命令，支持显式更新和执行引导仓库命令
  • 新增 opt-in 浮动注册表 (registry_floating)，可获取最新的注册表元数据
  • 为 cargo, npm, go, gem, pipx 和 dotnet 后端新增精确版本快速路径，跳过远程版本列表获取以提升安装速度
  • 支持 vfox Lua 插件作为系统包管理器，新增 [bootstrap.plugins] 配置及相关命令
  • 优化 Shell 激活流程，跳过重复的初始提示钩子，减少启动时的进程开销
  • 优化启动性能，减少配置查找时的 glob 模式编译，每次调用约节省 2ms
  • 通过 PGO 优化 linux-x64 和 macos-arm64 二进制文件，热路径性能提升约 10%
  • 优化后端存档提取和 macOS hdiutil/pkgutil 等待机制，避免并行安装时阻塞下载进度
  • 将 tokio 工作线程默认上限设为 16，避免多核机器上线程池过大
  • 统一 tar 提取与验证实现，移除系统 tar 回退，确保跨平台行为一致
  • 注册表更新：新增 zmx，docker-compose 切换至 github 后端，eza 切换至 vfox 后端

Breaking Changes

  • 配置验证更加严格：弃用 env 指令别名、env.mise 指令及实验性 monorepo root 键，非字符串 postinstall 钩子将被拒绝

问题修复

  • 修复远程 HTTP 和 Git 任务文件的 #MISE 标头解析问题，使其支持元数据覆盖
  • 修复全局或系统配置加载的文件任务未被正确视为全局任务的问题
  • 修复 mise —env watch 未能将选定环境传播至被监听任务的问题
  • 修复 HTTP 后端安装的原始可执行文件权限不足（改为 0755）导致无法打开输入文件的问题
  • 修复 Java 最新版本选择逻辑，优先选择基础供应商变体
  • 修复 bootstrap 中 systemd 定时器无法通过裸 TOML 键引用托管服务的问题
  • 修复 Homebrew Casks 声明 auto_updates 时导致安装失败的问题
  • 修复 macOS 默认值处理中缺失键或域导致初始化报错的问题
  • 修复 Windows 上 mise uninstall 未能清理悬空运行时版本指针文件的问题
  • 修复网络不可用时的快速失败机制，优化 DNS 解析失败处理和超时限制
  • 修复 dry run 模式下仍会产生副作用（如执行安装钩子）的问题
  • 修复多种后端版本检测问题，包括不带分隔符的预发布版本和 cargo 稀疏索引发现
  • 修复 JSON Schema 错误，提升编辑器对配置文件的验证准确性
  • 修复 lockfile 在处理平台特定工具选项变体时被折叠的问题
  • 修复多种配置验证问题，包括弃用 env 指令别名及拒绝非字符串 postinstall 钩子
