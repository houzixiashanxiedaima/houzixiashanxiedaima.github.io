code-yeongyu/oh-my-openagent v4.17.0 gpt-oss-120b

新内容

  • 新增基于任务规模和风险的 ULW 工作人员分层选择（低 / 中 / 高难度）
  • 默认安装模型升级为 GPT-5.6‑sol，具备 372K 上下文窗口
  • 为 6 Sol 使用 Luna 与 Terra 路由，实现更深度分析
  • ULW 循环支持自动恢复中断运行，加入 Stop Hook 限制
  • 任务 TUI 显示模型详情并宽度限制，终端控制序列安全清理
  • 新增难度分层工作代理并在 ULW 流程中进行委派指导
  • 插件安装改进，移除嵌套 MCP 清单的无效相对守护路径
  • 引入 spawn‑guard 与 reviewer preflight 检查，防止 fan‑out 失控
  • 证据按尝试绑定到对应提交，防止旧 HEAD 重新标记为证据
  • 升级依赖（lodash、axios 等），提升整体稳定性
  • 安装与发布工作流硬化，提升安全序列和可靠性

Breaking Changes

  • 移除旧的 lazycodex‑executor 代理，需更新工作流配置
  • 默认模型从 GPT-5 改为 GPT-5.6‑sol/372K，需适配新模型名称
  • 包检测、插件 / 配置兼容性及安装界面进行命名迁移，可能影响自定义脚本

问题修复

  • 修复 ULW 循环在 Windows CI 上的组件测试失败
  • 修复 worker 昵称字符集不符合 Codex 角色加载器要求
  • 删除插件包和缓存中的嵌套 .mcp.json 文件，防止路径错误
  • 修复 /stop‑continuation 在所有入口点的停止续写行为
  • 修复 ULW 循环的 Stop‑hook 延迟恢复逻辑
  • 若干文档修正（模型声明、代理列表、README 等）消除陈旧信息
  • 修复 ULW 循环尝试隔离分隔符兼容性并在 CI 中加入门控
  • 其他小错误修复（错误信息拼写、测试超时等）

JeoJay127/RapidEFI-Tool v5.2.0 gpt-oss-120b

新内容

  • 重构睡眠与唤醒 SSDT 处理逻辑，统一通过 SSDT‑SleepHook 调度，避免 _PTS、_WAK 重命名冲突
  • 改进 AMD 核显识别与 NootedRed 兼容性，并完善 Intel 核显代号识别，降低误判为独显的概率
  • Windows 发布包内置 Microsoft Visual C++ 运行库 DLL，解决未安装 VC++ 环境导致的启动失败

问题修复

  • 修复定制 SSDT 与预置 SSDT 重复写入问题，自动移除同名 ACPI 项，防止重复 AML 与加载异常
  • 修复不同 macOS 版本下 Intel 核显背光参数选择逻辑，正确处理 -igfxblr 与 -igfxblt 参数
  • 修复 SD 读卡器信息无法识别和展示的问题，优化硬件信息页面的 I/O 设备显示
  • 修复仅生成定制 SSDT 场景下 ACPI 输出目录不存在导致的合并输出不稳定
  • 修复 macOS 发布包符号链接错误，显著减小压缩包体积并保持 Framework 结构完整
