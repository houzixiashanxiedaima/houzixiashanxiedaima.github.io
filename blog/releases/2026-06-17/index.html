openclaw/openclaw v2026.6.8 GPT-OSS 120B

  • 修复渠道发送策略，保留账号范围 DM，改进 Telegram 富文本换行和表格
  • 修复 Discord 自动线程标题超时限制，设置 60 秒和 4096 token 上限
  • 修复 Agent、cron、Gateway 运行时，在重启前标记活跃会话并暂停被中止的子代理
  • 修复提供商和模型回放，保持无状态 OpenAI 响应兼容，恢复无效签名错误
  • 修复内存、状态、诊断配置，拆分大批量嵌入，避免 NFS 上 SQLite WAL
  • 修复 UI/mobile/TUI，保留仪表盘会话父链，改进 WebChat 回滚和侧边栏交互
  • 修复插件更新时缺失平台包，补全 Codex 平台二进制
  • 升级依赖 Hono 至 4.12.25，确保发布的 OpenClaw 包使用修补运行时
  • 提升发布和测试可靠性，延长网关看门狗，拆分本地全套分片，稳定插件认证

kepano/defuddle 0.19.0 GPT-OSS 120B

新内容

  • 新增对 Bilibili 视频字幕和转录的提取支持
  • CLI 新增 —frontmatter 标志用于输出 YAML 元数据
  • CLI 新增 —user-agent 标志并支持从 stdin 读取 HTML
  • 改进 Math 处理：重建 MathML 并优化复杂公式的 Markdown 转换
  • 默认在 Node 环境下异步提取 X（Twitter）文章，提高可靠性

问题修复

  • 解析时不再修改原始文档，避免副作用
  • 不再剥离没有分隔符的锚点 ID 所在的章节
  • 作者 / 日期行识别不再误将星期几当作日期
  • 修复 X/Twitter 非英文 aria-label 回复的提取
  • 修复 FxTwitter 面部索引在代理对表情符号时的错误

Breaking Changes

  • 移除 subhead 元素，可能影响依赖该结构的使用者
  • 从 SVG 中删除子元素以提升安全性，可能导致渲染差异

code-yeongyu/oh-my-openagent v4.11.0 GPT-OSS 120B

新内容

  • 将 CodeGraph 纳入默认代理工作流，提供更安全的 Node 运行时选择和可移植的 MCP 启
  • 新增监控功能，实现后台命令跟踪、会话感知输出注入及内存上限等硬化措施。
  • TUI 侧边栏优化项目镜像、标题安全、关闭行为并加入隐私脱敏。
  • Prometheus、ultrawork、ulw-loop 改进为基于路径的提示结构并强化质量门槛。
  • LazyCodex 与 Codex Light 达到发布成熟度，提供同步的市场负载和严格的插件元数据

问题修复

  • 不受支持的运行时现在能够平滑降级，避免启动崩溃。
  • Windows Git Bash 与安装程序行为得到改进，提升兼容性。
  • 通过更严格的包装检查修复了插件安装和挂钩问题。
  • 发行流水线现在验证主包和平台包，防止发布错误。
  • 输出过滤器的 ReDoS 硬化修复了潜在的拒绝服务风险。

Breaking Changes

  • Ast-grep 不再随内置 MCP 路径提供，改为通过共享 skill/sg 解析器获取。
  • 升级后需重启代理会话以加载新路径。
  • CodeGraph 与 TUI 侧边栏默认更突出，使用自定义配置的用户需检查被禁用的工具或 hook
