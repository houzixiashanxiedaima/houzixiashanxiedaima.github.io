steipete/CodexBar v0.40.0 gpt-oss-120b

新内容

  • 新增 Claude 只读账户卡片，堆叠显示且不延迟刷新
  • 新增 Claude 可直接在卡片切换非活跃账户并立即刷新
  • 新增 Codex 仪表盘显示今日及 30 天原始信用额度
  • 新增成本图表提供可见且安全的刻度标签
  • 新增 Cursor 在 Linux 读取已登录的应用令牌并支持手动 Cookie

问题修复

  • 修复 Codex 成本历史重复文件系统工作，提升缓存复用
  • 修复 Provider 检测 Claude Desktop，忽略无效 Gemini CLI
  • 修复 Claude 成本历史包含本地代理日志，保持覆盖范围
  • 修复菜单手动刷新仅作用于对应 Provider，避免其他行变灰
  • 修复语言选择器保持本地语言名称可读并遵循系统设置

hackjutsu/Lepton v2.0.0 gpt-oss-120b

新内容

  • 全部代码片段下载并支持全文搜索
  • 内置多种主题，支持 light、dark 等
  • 新增国际化（i18n）支持
  • 可设置 startAtLogin，实现系统启动自动运行
  • 支持窗口最小化或关闭至托盘

问题修复

  • 修复若干已知问题

Suwayomi/Suwayomi-Server v2.3.2238 deepseek-ai/DeepSeek-V4-Flash

新内容

  • 支持扩展 API v1.6 并迁移至扩展商店系统
  • 新增 SyncYomi 同步功能，兼容 Mihon 分支
  • OPDS 添加直接流 / 下载链接选项
  • 更新 H2 数据库引擎至最新版本
  • 使用最新 Chromium 内核的 JCEF 渲染

问题修复

  • 修复扩展未安装时错误显示更新提示
  • 修复下载后重命名漫画导致文件丢失
  • 修复章节列表更新失败时数据丢失
  • 修复 PostgreSQL 下漫画排序查询错误
  • 修复 Windows Java 上 chmod 执行失败

Breaking Changes

  • 扩展仓库可能因新商店系统丢失，需备份并重新添加

ejbills/DockDoor 1.39.4 gpt-oss-120b

新内容

  • 将原生标签页折叠为单个预览
  • 支持 Safari、Finder、Terminal、Ghostty 等标签式应用每窗口仅显示一个预览
  • 新增玻璃 Variant 控件，可通过滑块选择不同的玻璃外观
  • 预览更平稳，减少闪烁并一次性完成渲染
  • 将文件夹预览选项迁移至 Dock 预览标签页

问题修复

  • 修复视图切换时误退出应用的问题
  • 在启用 “关闭最后一个窗口时退出” 时，重新检查 Office 与 Chrome 应用防止中途关闭
  • 媒体小部件的播放 / 暂停响应更可靠
  • 按钮不再在暂停 / 恢复时闪烁或导致时间跳动
  • 关闭或失效的窗口不再在预览中残留

JanDeDobbeleer/oh-my-posh v29.20.1 - v29.21.0 gpt-oss-120b

新内容

  • 在 PowerShell 中支持 vimode 段
  • 默认使用原生毫秒计时
  • 跳过在传入参数时的 mousetrap 检查
  • 缓存颜色解析以提升渲染速度
  • 并发执行所有块段以加速提示生成
  • 延迟创建 Windows 作业对象以降低启动开销
  • 缓存模板解析并避免二次补丁

问题修复

  • 共享缓存文件句柄并防止锁竞争时截断
  • 原子写入 Unix 缓存文件
  • 修复多行命令的错误
  • PowerShell 流式输出时避免渲染前一行
  • PATH 变化时使持久化命令查找失效

ythx-101/x-tweet-fetcher v3.0.0 gpt-oss-120b

新内容

  • 完全重构，保持相同 CLI，内部实现全新
  • 将 scripts/ 单体改为可通过 pip 安装的包，提供 xtf 命令和 Router 模块

Breaking Changes

  • 统一 —search per-tweet schema 与 —user，author/autho
  • 仓库范围仅限推文获取，分析、China-platform、Obsidian 脚本已删除，旧功能保留在
  • 不再硬编码公共 Nitter 默认，需通过 XTF_NITTER 环境变量自行配置

nguyenphutrong/quotio v0.21.0 - v0.22.0 gpt-oss-120b

新内容

  • 提供 DMG 安装包下载

Breaking Changes

  • 未使用 Apple 开发者证书签名，需手动允许运行
  • 应用未签名，macOS 可能阻止运行，需手动执行命令绕过

Homebrew/brew 6.0.7 gpt-oss-120b

新内容

  • 在 Linux 上将 CC 和 CXX 设置为未版本化的 gcc
  • 强制 CMake 在 $HOMEBREW_PREFIX/bin 前查找 shims
  • 允许在容器中运行 bubblewrap
  • 在依赖展开期间缓存公式查找，提高性能
  • 限制 livecheck 与 curl 重定向，提升安全性

问题修复

  • 修复 audit 中 curl 依赖缺少可用 HTTP 镜像的问题
  • 修复 bump-cask-pr 在更新失败时的错误处理
  • 修复 contributions 中的类型错误
  • 修复 bump 中无效架构导致的问题
  • 修复 style 中数组字母排序的无限循环

rorkai/App-Store-Connect-CLI 2.6.0 gpt-oss-120b

新内容

  • 新增 Web 应用删除命令
  • 新增 Web 已删除应用列表命令

OthmanAdi/planning-with-files v3.3.0 - v3.4.0 gpt-oss-120b

新内容

  • 新增每次调用可选关闭钩子的变量
  • 关闭后不再注入计划上下文
  • 关闭后不写入计划文件
  • 仍保留工具调用的允许决策
  • 完整覆盖所有 Codex 钩子入口
  • 新增在草稿计划审阅期间即可启动计划注入、工具前朗读、写后提醒和自动继续
  • 引入显式确认步骤，需使用 /plan-execute 才会激活计划
  • 扩展在批准前仅显示状态行，保持被动模式
  • 确认范围限定于当前会话和计划路径，随会话生命周期重置
  • 持久化的 markdown 文件保留在磁盘上

问题修复

  • 修复一次性会话被钩子劫持的问题

Breaking Changes

  • 扩展不再在 task_plan 时立即激活所有钩子，需显式确认后才会触发
