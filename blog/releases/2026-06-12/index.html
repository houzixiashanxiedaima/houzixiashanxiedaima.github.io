jdx/mise v2026.6.3 - v2026.6.4 GPT-OSS 120B

新内容

  • 新增声明式系统包支持（apt、dnf、pacman、Homebrew）并提供 experimenta
  • 新增 mise system use 与 mise system upgrade，可写入 syste
  • 为 github、gitlab、forgejo 后端加入 matching 与 matching_r
  • mise oci build 支持 —owner UID [:GID]，并可通过 oci.user_
  • 注册表新增 ghtkn 与 antigravity-cli 两个工具
  • 新增 auto_env 早期初始化设置，可自动激活平台相关配置环境
  • 新增 GitHub token 刷新命令，强制获取新 OAuth 令牌
  • 版本表达式支持后处理正则结果，实现更精准的版本解析
  • 在 mise deps 中加入对 Deno 的完整支持，支持检测、安装等操作
  • 注册表新增 imagemagick 与 vscode-cli 包

问题修复

  • —minimum-release-age=0s 现可正确禁用发布时间限制，避免不必要的完整版本列表
  • 修复 Aqua 工具的 GitHub 元数据路由，仅在 mise-registry 支持时生效，消除
  • 任务引用解析在验证阶段统一处理，解决误报缺失依赖的问题
  • Windows 上转发参数现在可正确传递给 inline bash -c 任务
  • 改进编辑器 (EDITOR / VISUAL) 命令处理，支持带参数的值并提升错误报告
  • 修复默认 24h minimum_release_age 导致启动慢的问题
  • 修复 minimum_release_age 为 “0s” 时未禁用的错误
  • 为 Python 远程版本添加发布时间戳，确保过滤生效
  • 修复工具解析 .node-version 等文件写入错误锁文件的问题
  • 修复 Windows 任务内联和钩子命令双引号转义问题

openclaw/openclaw v2026.6.6 GPT-OSS 120B

问题修复

  • 修复会话恢复时丢弃过期批准、清理耗尽的回复队列、恢复陈旧回复
  • 修复内容边界，屏蔽协议产物并隐藏转录图片
  • 修复渠道投递，重启后保留 WhatsApp 回复并恢复 Discord、Mattermost、Fei
  • 修复定时任务，干净取消活动任务并保留超时状态
  • 修复 Android，避免持久节点使用 dataSync 前台服务类型

  • 重新实现 iOS 必要的终端和通知修复
  • 修复 iOS 重新排列工作区标题时的显示问题

anomalyco/opencode v1.17.4 GPT-OSS 120B

新内容

  • 新增对本地 MCP 服务器的 cwd 支持，可从工作区相对目录启动
  • 新增基于连接器的认证流程并支持存储的提供者凭证
  • 新增 v2 API 端点用于创建 / 获取会话、列出会话问题以及解析活动位置
  • 文件读取现在返回原始内容并提供正确的 Content-Type，供 API 与 SDK 使用
  • MCP 服务器日志通知现已在 opencode 日志中显示

问题修复

  • 修复 Gemini 工具模式使用多类型字段的问题，保持兼容工具正常工作
  • 内容过滤的模型响应现在显示为可见错误，而不再静默失败
  • 修复 MCP 认证和调试请求丢失已配置的请求头
  • 快照创建现在复用源 Git 对象，避免在大型仓库上长时间重新哈希
  • 修复 MCP catalog 请求超时在列出提示、资源或工具时未生效的问题

code-yeongyu/oh-my-openagent v4.9.2 GPT-OSS 120B

新内容

  • 新增可靠的后台代理唤醒路由，通过实时服务器路径直接唤醒父会话
  • 支持每实例路由作用域，防止多实例 serve () 之间的跨实例干扰
  • 在插件入口自动注册实时服务器路由，实现 fire-and-forget
  • 将父目标调度通过实时监听客户端路由，实现异步门控
  • 提供 experimental.disable_live_parent_wake_routing 配

问题修复

  • 通过同步快速路径解决方案，消除唤醒信号的竞争条件
  • 修复后台任务完成后唤醒信号可能丢失的死锁问题
  • 修正实时服务器路由的调度顺序保持
  • 为多实例 serve () 注册隔离路由状态，避免跨实例冲突
  • 修复后台代理在忙碌守护读取时的刷新问题

Breaking Changes

  • 默认启用新实时父会话唤醒路由，若需保持旧行为请通过实验配置关闭
