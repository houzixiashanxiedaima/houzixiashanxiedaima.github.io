OthmanAdi/planning-with-files v3.8.0 - v3.8.1 gpt-oss-120b

新内容

  • 计划解析不再依赖实时 shell 工作目录
  • 解析锚点改为最近的祖先目录，受仓库边界和深度限制约束
  • 显式 PLAN_ID 在任意子目录仍可使用
  • 每次 Pi 注入显示解析的计划路径
  • 规划目录不再遮蔽根 task_plan
  • md 标签可见性改为显式而非沉默
  • 初始化会话创建计划时包含 v3
  • 修复 Next Step 部分未写入真实计划的问题
  • 修复 26 处 heredoc 复制并加入回归测试
  • 升级捆绑的 Pi 扩展至 1 版
  • npm 包 @tomxprime/planning-with-files 同步发布
  • 加强安全硬化：解析器匹配 sh 解析器的 slug 验证与 containment
  • 新增 vitest 覆盖 35 项测试，提升可靠性
  • 新增可选结构感知注入（PWF_INJECT=smart），在长计划窗口中显示标题、目标、下一步等信息
  • 默认输出保持与 v2 完全一致的字节内容
  • 计划模板加入 “下一步” 章节和第六个重启问题
  • 会话捕获报告中加入工具结果输出
  • 添加 macOS 最新 CI 任务及 BSD 用户空间仿真环境
  • 新增 .gitattributes、三篇问题查询文档以及重新编写的 LLM 模块
  • 更新 README，加入 SVG 基准图表并嵌入内部基准框架
  • 测试套件通过 301 项（比 217 项提升），在 Windows、Linux 上绿色，macOS 待运行

问题修复

  • 修复 Stop hook 在 macOS、Linux 上未触发的问题
  • 修复调度相关的两个 bug：安装路径回退使用不可替换的 shell 默认，PowerShell 分支错误选择
  • 修复缺少 PowerShell 时 ps1 脚本导致的退出码 127
  • 修复完成建议和 v3 完成门在部分环境下失效的问题
  • 修复标量选择目标时依据文件存在性并按平台调度的错误
  • 修复会话恢复搜索路径错误以及 /clear 后恢复未找到内容的静默问题
  • 修复 claude/projects 目录在 macOS、Linux 安装以及包含下划线的项目路径中的处理错误
  • 修复映射器先检查精确拼写并保留旧拼写作为回退的逻辑
  • 修复陈旧 SHA 缓存导致错误的 PLAN TAMPERED 报告（缓存键加入项目根目录）
  • 修复 ps1 与 sh 解析器在 slug 验证、task_plan、md 要求、ledger-append 等方面的兼容性
  • 修复 sh 在处理 CJK 或表情符号摘要时截断问题

ogulcancelik/herdr v0.7.5 gpt-oss-120b

新内容

  • 新增 live-agent CLI，支持命名启动、原子提示、逻辑发送键和服务器拥有的等待工作流
  • 新增瞬态声明式 Agent 视图查询（agent.view.set/clear），并定义侧边栏、移动端、鼠标、键绑定的排序顺序
  • 新增一次性插件 [[startup]] 钩子，用于在服务器启动和实时交接后恢复插件状态
  • 为展开的 Space 与 Agent 侧边栏行布局添加 per‑token 前景色、粗体和暗淡样式
  • 新增 ui.sidebar_start_collapsed 配置，启动时侧边栏默认折叠
  • 新增 ui.prompt_new_workspace_name，创建交互式 TUI 前询问工作区名称
  • 新增 macOS 对 HERDR_AGENT= 前景进程提示的支持，允许隐藏在可见包装器后的代理使用命名代理的屏幕清单
  • 改变：Agent 命令仅接受唯一的 live agent 名称或所在 pane ID…
  • 改变：会话导航器使用连接树图标，按工作区分组匹配，并在搜索开始时自动选中第一个结果

Breaking Changes

  • 插件的安装和链接状态改为全局用户级别，之前仅在特定 Herdr 会话中安装的插件需重新安装或链接

问题修复

  • 修复 CLI 请求在协议不匹配时返回 machine‑readable 错误，并改进 agent wait 超时返回行为
  • 修复 Linux 声音通知未能终止 mpg123 进程导致的 CPU 占用问题
  • 修复超大括号文本粘贴被拒绝、单独 Escape 键被错误合并以及粘贴模式下的提示行为
  • 修复侧边栏、工作目录缓存、终端 diff 流批处理、窗口大小调整后视口跟随等 UI 渲染性能问题
  • 修复 Codex、Grok、OpenCode、Kimi、Pi 等代码集成在信号、阻塞状态以及生命周期报告上的错误
  • 修复 Windows 上插件、集成、前景进程快照、非交互子进程窗口闪烁以及远程附加路径发现等兼容性问题
  • 修复插件命令数组保留仅空白参数、插件在无服务器时仍可安装 / 链接以及实时交接保留已安装插件等功能缺陷

  • 修复 Moonshot AI 系列的 Kimi K3 模型使用 OpenAI 思考格式并支持推理耗费

code-yeongyu/oh-my-openagent v4.19.1 gpt-oss-120b

新内容

  • GPT 默认模型升级至 GPT-5.6 Sol，自动路由无需配置
  • 启动时进行家族式进程清理，父进程监控看门狗防止孤儿进程
  • LSP 守护进程在启动时回收旧版守护进程
  • CodeGraph 升级至 1.4.1 并提供可选共享守护进程模式 OMO_CODEGRAPH_DAEMON
  • LSP 诊断更精准：静默服务器不再因新鲜度超时阻塞，准确上报取消和超时请求
  • ulw-loop 批处理效率提升：原子批量引导、验证批次边界、目标自动前进合并检查点
  • ulw-plan 强制打开声明，停止恢复写入限制在会话状态目录
  • Windows 与 Codex 安装可靠性增强：bun test 在 win32 正常退出，cmd shim 支持 doctor 启动…
  • 模型路由修正：antigravity‑gemini‑3‑flash ID 不再错误重写，sol/terra/luna 模型拥有 372k 上下文预算…
  • 自动在计划提交时生成记事本文件（start‑work 自动 scaffold）

Breaking Changes

  • 移除旧版模型回退机制，若依赖自动回退需更新配置或模型选择

问题修复

  • 修复模型路由相关错误，包括 ID 重写、回退层级及上下文预算问题
  • 解决 Windows CI 计时不稳定及合并刷新等待竞争问题
  • 修复 npm‑ci 锁漂移导致的安装失败
  • 解决 Codex 仅安装时退出状态不正确的回归
  • 消除 duplicate agent role 警告并避免角色重复注册
  • 修复 marketplace 根目录 manifest 写入及组件 CLI 打包问题
  • 解决 TypeScript 在调用项目中的解析错误
  • 修复 LSP 诊断在无拉取服务器上的超时行为
  • 解决 ulw‑plan 与 ulw‑loop 的冲突及检查点自动前进问题
  • 其他若干小修复（如文档更新、测试不稳定等）
